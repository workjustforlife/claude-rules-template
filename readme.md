## 🤖 新项目 Claude Code 配置规划指南

为了保证 AI（Claude Code）在开发本工程时能够完美理解中英文环境、严格控制代码质量并遵守硬件/前端规范，在新项目初始化时，请根据项目类型在根目录下配置对应的规则文件：

### 1. 规则文件选型流水线（三选一）
依据当前工程的技术栈，从以下三大完整模板中复制对应内容，保存为项目根目录下的 **`CLAUDE.md`** 并提交至 Git：
- **纯底层驱动/C++ 项目**：使用《C/C++ 与嵌入式 Linux 专属局部规范模板》，重点约束内存零泄漏、驱动解耦和 Makefile 变量编译。
- **UI 交互项目**：使用《Qt/QWidget 跨平台开发局部规范模板》，重点约束对象树自动释放、C++11 信号槽语法、以及强行剥离耗时任务至 `QThread` 防止 UI 卡死。
- **前端网页/Web 项目**：使用《大一统前端通用开发与测试规范模板》，重点约束 DOM 异步防御、严格类型检查、Playwright 自动化测试以及 `live-server` 本地预览构建。

### 2. 宿主机本地私密配置（每台机器必做）
不同宿主机、不同开发板的硬件环境与路径各不相同。为了避免环境冲突或泄露个人路径：
1. **必须**在项目根目录的 `.gitignore` 文件中追加一行：`CLAUDE.local.md`。
2. 在本地创建 **`CLAUDE.local.md`** 文件（不提交 Git），填入当前机器的专属环境参数：
   - 交叉编译器与工具链的绝对路径（如 `CC="..."`、`CXX="..."`）。
   - 当前测试板的本地串口号（如 `/dev/ttyUSB0`）与网络 IP 地址。
   - 自定义高频自动化口令（如 `deploy`：实现一键编译、scp 推送、ssh 赋权运行并抓取日志）。

### 3. Git 提交双重审查机制（所有项目通用）
在所有项目的 `CLAUDE.md` 末尾，必须保留以下交互死命令，以确保代码入库的绝对安全：
- 强制 AI 生成英文的约定式提交规范（Conventional Commits，如 `feat: ...`, `fix: ...`）。
- **严禁擅自 Commit**：AI 绝对不允许直接运行 `git commit -m` 强行提交。
- **引导用户审查**：AI 生成完 Commit 文本后必须踩刹车，停下来引导用户：“请审查以上提交信息。如果没有问题，您可以手动运行终端命令或让我继续。”

### Mac / Linux / WSL 终端一键绑定命令：
打开终端，进入本仓库所在的目录，执行以下命令：
```bash
# 创建 Claude 全局配置目录
mkdir -p ~/.claude

# 建立软链接，将本地全局配置直接指向本仓库中的全局模板
# (请确保运行命令时处于当前仓库根目录下)
ln -sf "$(pwd)/global/CLAUDE.md" ~/.claude/CLAUDE.md

echo "🎉 Claude Code 全局软链接配置成功！"
```
