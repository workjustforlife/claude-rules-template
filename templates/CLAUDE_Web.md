# 前端项目通用开发与交互规范 (HTML/CSS/JS & React/Vue/TS)

## 规范执行优先级
- 本文件中的规则适用于当前项目的所有网页结构搭建、样式编写、组件开发、逻辑调试以及自动化测试。
- **项目技术栈感知**：在动笔修改任何代码前，必须先阅读 `package.json` 或项目目录结构。如果是简单网页，严格使用纯原生（Vanilla）方案；如果是 React/Vue 项目，必须适配框架的官方推荐写法，保持代码的纯粹与一致性。
- **最小修改原则**：只修改实现功能所需的必要代码，严禁为了重构而大幅改动无关的组件或业务逻辑。

# 核心技术规范

## 1. 原生网页基础与 UI 样式规范
- **语义化结构**：编写 HTML 时结构必须清晰，优先使用语义化标签（如 `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`），避免通篇滥用 `<div>`。
- **外部资源引入**：纯 HTML 项目中，外部 CSS 文件必须在 `<head>` 中引入，外部 JS 文件必须在 `<body>` 底部引入，或使用 `defer` 异步延迟加载，确保首屏加载速度。
- **严禁乱写行内样式**：样式必须写在独立的 CSS 文件或组件样式作用域内。**绝对不允许**直接在 HTML 标签上硬编码 `style="..."`（除动态 JS 计算值外）。
- **现代布局优先**：实现页面布局和居中时，**必须优先使用 Flexbox（弹性盒）或 CSS Grid（网格）布局**。坚决废弃过时的 `float` 浮动布局或大面积的 `absolute` 绝对定位强行对齐。
- **命名规范**：CSS 类名（Class Name）必须使用语义化的英文小写加中划线命名（例如 `.btn-submit`、`.nav-item`），严禁使用拼音或无意义命名（如 `.div1`、`.aa`）。

## 2. 框架开发、类型安全与状态管理 (若项目适用)
- **严格类型检查 (TS)**：在 TypeScript 项目中，严格遵守 `tsconfig.json` 配置，**严禁滥用 `any` 类型**。复杂数据必须明确定义 `interface` 或 `type`。
- **现代 JS/TS 语法**：优先使用现代化 ESNext 语法（如解构赋值、可选链 `?.`、空值合并 `??`、箭头函数等）。
- **单一职责组件**：组件设计要做到高内聚、低耦合。一个组件文件原则上只做一件事，复杂的页面应当合理拆分为基础组件（Atom）与业务组件（Feature）。
- **Hooks/组合式 API 规范**：
  - React 项目：严格遵守 Hooks 规则，`useEffect` 等依赖项数组必须写全，严禁引发无限循环。
  - Vue 项目：优先使用 `<script setup>` 组合式 API 语法。

## 3. DOM 交互、异步防御与安全 (最高优先级)
- **DOM 异步防御**：在执行任何原生 DOM 操作前，必须确保 DOM 已经加载完毕（包裹在 `DOMContentLoaded` 中），或者确保被操作的元素绝对存在，防止报 `Cannot read properties of null` 错误。
- **副作用隔离与清理**：所有网络请求、定时器、本地存储（LocalStorage）等副作用操作，在 React/Vue 中必须在相应的生命周期钩子中进行，且**必须包含完善的清理逻辑**（如 `clearTimeout` 或取消未完成的请求）。
- **性能优化 (防抖与节流)**：对于高频触发的事件（如 `resize`, `scroll`, `input`），必须配合标准的防抖（Debounce）或节流（Throttle）函数处理。
- **XSS 安全防御**：在向网页动态插入用户输入的文本时，**严禁使用 `innerHTML` 或 `v-html`**，必须使用 `textContent`、`innerText` 或双大括号语法（`{{ }}`）逃逸特殊字符，防止跨站脚本攻击。

# Web 自动化测试规范 (Testing)

## 1. 测试工具与技术栈选型
- **单元与逻辑测试 (Unit Test)**：若项目涉及复杂的 JS 纯逻辑函数、数据处理算法，优先使用 `Jest` 或 `Vitest` 编写断言测试。
- **UI 与端到端测试 (E2E Test)**：若项目需要测试用户交互逻辑（如：点击按钮、提交表单、页面跳转），优先使用现代化的 `Playwright` 或 `Cypress` 编写端到端自动化脚本。
- **原生小项目 (Vanilla JS)**：若是纯原生网页，且未引入复杂构建工具，优先使用原生 `console.assert()` 编写轻量级内联测试，或者利用 Node.js 自带的 `node:test` 模块。

## 2. 测试用例编写质量
- **用户视角优先**：编写 UI 自动化测试时，**严禁硬编码依靠类名（Class Name）或标签来定位元素**。必须优先通过用户可见的文本或可访问性标签来选择（例如使用 `page.getByRole('button', { name: 'Submit' })` 或 `page.getByText('确认')`），确保测试的健壮性。
- **三段式结构 (AAA 原则)**：每个测试用例必须结构清晰：包含 Arrange（准备数据/Mock）、Act（模拟用户操作）、Assert（断言验证结果）三个清晰步骤。

## 3. 异步测试防御（极重要）
- **杜绝硬编码等待**：在等待异步请求、弹窗动画或页面加载时，**绝对不允许**使用硬编码的死等时间（如 `setTimeout(..., 3000)`）。
- **智能等待机制**：必须使用测试框架提供的智能等待接口（如 Playwright 的 `expect(locator).toBeVisible()` 或 Jest 的 `waitFor()`），依靠 DOM 的状态动态向下执行，避免由于网络波动导致测试不稳定（Flaky Tests）。
- **接口 mock 防御**：端到端测试中，为了防止后端接口不稳定影响测试结果，在测试网络请求时，必须使用测试框架拦截并 mock 对应的 API 返回（如使用 `page.route()` 拦截 AJAX 请求），确保测试可在离线/独立环境下 100% 成功跑通。

# 项目本地预览与构建规范 (Build & Run)

## 1. 纯原生项目本地预览 (Vanilla JS Live Server)
- **避免直接双击 HTML**：对于纯原生网页，严禁直接双击 `index.html` 用 `file://` 协议预览。这会导致模块化脚本（`type="module"`）和异步请求（Fetch/AJAX）因跨域策略（CORS）报错。
- **智能开启本地服务器**：当我要求你“运行网页”、“启动预览”或“查看效果”时，请自动根据当前宿主机环境选用以下非阻塞命令之一启动本地轻量级 Web 服务器：
  - 优先检查并使用 Node.js 工具：`npx live-server` 或 `npx http-server`
  - 若无 Node 环境，则使用 Python3：`python3 -m http.server 8080` 或 `python -m http.server 8080`
- **提示访问路径**：启动成功后，用中文明确告诉我本地访问的 URL 地址（例如 `http://localhost:8080`）。

## 2. 现代框架构建规范 (React/Vue/Vite/Webpack)
- **包管理器对齐**：通过锁定文件（如 `package-lock.json`、`pnpm-lock.yaml`）自动识别当前项目使用的包管理器。安装新依赖时必须使用与之匹配的命令（如 `pnpm add`），严禁混用包管理器。
- **标准构建命令**：在带有构建打包工具的项目中，优先通过 `package.json` 中的 `scripts` 脚本执行构建。统一使用非交互式命令（如 `npm run build`、`pnpm build`），并带上参数防止终端挂起。
- **零构建污染原则**：所有打包产物必须输出到独立的发布目录（如 `dist/`、`build/`、`out/`），且该目录**必须**被拉入 `.gitignore` 彻底忽略，严禁将编译产物推送到源代码仓库。
- **构建错误自查**：如果在执行 `build` 过程中终端报错（如打包失败、Linter 报错或 TS 类型编译失败），你必须**自动截获终端日志并进行分析**，直接定位到源码中的错误并进行修复，直至项目能 100% 成功打包。

# 团队协作与沟通

## 交互习惯
- **语言偏好**：与我（用户）交谈、解释代码、汇报进度时，全程使用清晰、专业的中文。
- **代码与 Git 规范**：代码内的注释、组件命名、变量命名以及 Git 提交信息（Commit Message），一律使用**英文**编写。
- **提交规范**：Git Commit 必须遵守约定式提交规范（Conventional Commits，例如：`feat: ...`, `fix: ...`）。
- **严禁擅自 Commit**：生成完规范的提交信息后，你必须停下来，**绝对不允许**直接运行 `git commit` 命令。请用中文提示我：“请审查以上提交信息。如果没有问题，您可以手动运行终端命令或让我继续。”
